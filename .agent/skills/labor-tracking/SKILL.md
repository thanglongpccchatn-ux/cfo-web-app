# Goal

Hướng dẫn nhập và theo dõi các đợt thanh toán thầu phụ/tổ đội, bao gồm giá
trị hợp đồng thầu phụ, khối lượng hoàn thành, đề nghị thanh toán và thực tế
đã chi trả theo từng đợt.

# Instructions

## Cấu trúc 1 dòng thầu phụ (Excel-like):

| Field | Mô tả |
|-------|-------|
| Thầu phụ/Tổ đội | Tên nhà thầu hoặc tổ thi công *(BẮT BUỘC)* |
| Đợt thanh toán | VD: "Tạm ứng", "Đợt 1 KL hoàn thành", "Quyết toán" *(BẮT BUỘC)* |
| Giá trị HĐ & PLHĐ | Giá trị hợp đồng/phụ lục thầu phụ ký kết |
| Ngày đề nghị | Ngày thầu phụ gửi ĐN thanh toán |
| GTHT đến cuối kỳ trước | Khối lượng lũy kế hoàn thành đến kỳ trước |
| GTHT kỳ này | Khối lượng hoàn thành trong kỳ hiện tại |
| Giá trị đề nghị TT | Số tiền thầu phụ đề nghị (chưa duyệt) |
| Giá trị tối ưu TT | Số tiền được duyệt thực tế *(QUAN TRỌNG)* |
| Ngày thanh toán | Ngày thực chi |
| Thực tế TT | Số tiền thực tế đã chi |
| Mức độ ưu tiên | Cao / Bình thường / Thấp / Khẩn cấp |

## Quy trình nhập:

1. Nhấn **"THÊM DÒNG MỚI"** (nút tím)
2. Điền Tên thầu phụ + Đợt thanh toán (BẮT BUỘC)
3. Nhập giá trị theo thứ tự: HĐ → Ngày → GTHT trước → GTHT kỳ này → Đề nghị → Phê duyệt → Thực tế
4. Chọn mức độ ưu tiên (Cao/Khẩn cấp nếu thầu phụ đang bị nợ lâu)
5. Nhấn ✅ Lưu

## Tổng hợp số liệu (header bar):
- **Tổng T.Ưu Cần Trả** = Σ `approved_amount` (tổng đã được duyệt trả)
- **Đã Thanh toán** = Σ `paid_amount` (tiền thực chi)
- **Còn Nợ Thầu Phụ** = Còn Trả − Đã Chi

# Examples

## Ví dụ 1: Nhập đợt tạm ứng thầu phụ Minh Đức

**User:** "Tạm ứng cho thầu phụ Minh Đức 500 triệu, HĐ thầu phụ 2 tỷ, đã trả 300 triệu ngày 10/3"

| Field | Giá trị |
|-------|---------|
| Thầu phụ | Minh Đức |
| Đợt | Tạm ứng |
| Giá trị HĐ | 2,000,000,000 |
| Ngày đề nghị | (ngày nộp đề nghị) |
| Giá trị đề nghị | 500,000,000 |
| Giá trị tối ưu TT | 500,000,000 |
| Ngày TT | 10/03/2026 |
| Thực tế TT | 300,000,000 |
| Ưu tiên | Cao |

## Ví dụ 2: User hỏi "Khối lượng hoàn thành là gì?"

**Trả lời:**
- **GTHT đến cuối kỳ trước**: Tổng KL thầu phụ đã làm từ đầu đến hết kỳ before
- **GTHT kỳ này**: KL làm mới trong kỳ hiện tại
- **Giá trị đề nghị TT**: = GTHT kỳ này × đơn giá thầu (thường do thầu phụ tự tính)
- **Giá trị tối ưu TT**: Kế toán điều chỉnh/duyệt có thể nhỏ hơn đề nghị

# Constraints

- 🚫 KHÔNG lưu nếu thiếu Tên thầu phụ hoặc Đợt thanh toán
- ✅ "Thực tế TT" ≠ "Tối ưu TT" — thường trả từng phần, không trả hết một lúc
- ✅ "Còn nợ thầu phụ" = Σ approved_amount − Σ paid_amount
- 📌 DB table: `expense_labor`
- 🔑 Priority values: `"Cao"`, `"Bình thường"`, `"Thấp"`, `"Khẩn cấp"`
